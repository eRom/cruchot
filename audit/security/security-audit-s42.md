### [H-01] YOLO Bash: No Command Blocklist + Shell String Injection
**Severity**: HIGH | **Priority**: P1 | **Source**: [MANUAL]
**File**: `src/main/services/seatbelt.ts:84,97-110`
**OWASP**: A03:2021 — Injection

**Description**: The `execSandboxed()` function had two issues: (1) On macOS, it used `execAsync` with string interpolation to build the shell command, creating a shell injection vector even with single-quote escaping. (2) On non-macOS platforms, it fell back to raw `execAsync(command)` with zero filtering — no blocklist at all. The workspace-tools bash has ~39 blocked patterns; YOLO bash had none.

**Impact**: On Linux/Windows, the LLM had completely unrestricted shell access. On macOS, edge cases in shell quoting could allow command injection.

**Fix Applied**:
1. Added `YOLO_BLOCKED_PATTERNS` (12 critical patterns: newline, backtick, `$()`, heredoc, sudo, su, etc.)
2. Replaced `execAsync` (string-based) with `spawn` (argument-array) — eliminates shell interpolation entirely
3. Blocklist applies to ALL platforms before execution

**Validation**: Semgrep re-scan clean (FP `detect-child-process` on intentional spawn).

### [H-02] sandbox:activate Accepts Arbitrary workspacePath Without Validation
**Severity**: HIGH | **Priority**: P1 | **Source**: [MANUAL]
**File**: `src/main/services/sandbox.service.ts:19-28`, `src/main/ipc/sandbox.ipc.ts:9`
**OWASP**: A01:2021 — Broken Access Control

**Description**: The `createSession(workspacePath)` method accepted any string as the sandbox directory. A compromised renderer could supply `/` or `/etc` as `workspacePath`, causing all YOLO tools to operate on system directories.

**Impact**: Path traversal / privilege escalation. The entire filesystem becomes writable by the LLM.

**Fix Applied**: Added `validateWorkspacePath()` with:
- `BLOCKED_SANDBOX_ROOTS` (15 system paths for macOS/Windows/Linux)
- `UNSAFE_PATH_CHARS` regex (blocks SBPL injection characters)
- Minimum 2 path segments (prevents `/` or `C:\`)
- `realpathSync` symlink resolution
- Absolute path requirement

### [H-03] SBPL Profile Injection via sandboxDir Special Characters
**Severity**: HIGH | **Priority**: P1 | **Source**: [MANUAL]
**File**: `src/main/services/sandbox.service.ts:60-66`
**OWASP**: A03:2021 — Injection

**Description**: The Seatbelt profile was generated by direct string replacement of `${SANDBOX_DIR}` into SBPL expressions. If `sandboxDir` contained parentheses, quotes, or newlines, it could break the SBPL syntax and inject arbitrary sandbox rules.

**Impact**: Complete Seatbelt bypass — the LLM could read/write any file on the system.

**Fix Applied**:
1. `sandboxDir` and `home` are sanitized: `replace(/["\n\r\t\0()]/g, '')` before SBPL injection
2. Removed trailing `deny` rules that were redundant with `(deny default)` and could cause rule-ordering ambiguity

### [H-04] validatePath Parent Check Missing path.sep
**Severity**: HIGH | **Priority**: P1 | **Source**: [MANUAL]
**File**: `src/main/llm/yolo-tools.ts:22`
**OWASP**: A01:2021 — Broken Access Control

**Description**: The parent directory check used `realParent.startsWith(sandboxDir)` without appending `path.sep`, meaning `/sandbox-abc-escape/` would be accepted if sandboxDir was `/sandbox-abc`.

**Fix Applied**: Changed to `!realParent.startsWith(sandboxDir + path.sep) && realParent !== sandboxDir`

### [H-05] openPreview Allows Dangerous File Extensions
**Severity**: HIGH | **Priority**: P1 | **Source**: [MANUAL] + [ELECTRONEGATIVITY]
**File**: `src/main/llm/yolo-tools.ts:193-198`, `src/main/ipc/sandbox.ipc.ts:119`
**OWASP**: A01:2021 — Broken Access Control

**Description**: `shell.openExternal(file://...)` was called without checking the file extension. The LLM could create a `.command`, `.sh`, `.app`, or `.scpt` file and open it — which macOS executes automatically.

**Fix Applied**: Added `DANGEROUS_PREVIEW_EXTENSIONS` blocklist (30 extensions: .app, .exe, .command, .sh, .scpt, .pkg, .dmg, etc.) in both `yolo-tools.ts` and `sandbox.ipc.ts`.

---

### [H-06] Missing setPermissionRequestHandler
**Severity**: HIGH | **Priority**: P1 | **Source**: [ELECTRONEGATIVITY] + [MANUAL]
**File**: `src/main/index.ts`
**OWASP**: A05:2021 — Security Misconfiguration

**Description**: No `setPermissionRequestHandler` was registered. The default Electron behavior grants ALL permission requests (camera, microphone, geolocation). A compromised renderer could capture audio/video or track location.

**Fix Applied**: Added `session.defaultSession.setPermissionRequestHandler((_wc, _perm, cb) => cb(false))` — denies all permissions.

### [M-02] shell.openExternal Called with Raw URL in window.ts
**Severity**: MEDIUM | **Priority**: P2 | **Source**: [ELECTRONEGATIVITY] + [MANUAL]
**File**: `src/main/window.ts:50`

**Description**: Trusted domain URLs were passed raw to `shell.openExternal()`. URL manipulation attacks (e.g., `https://github.com\attacker.com`) could trigger unexpected OS behavior.

**Fix Applied**: URL is now reconstructed from parsed components before `shell.openExternal`.

### [M-03] Processes Never Tracked — killAll is a No-Op
**Severity**: MEDIUM | **Priority**: P2 | **Source**: [MANUAL]
**File**: `src/main/services/seatbelt.ts`, `src/main/services/process-manager.service.ts`

**Description**: `processManagerService.track()` is never called from `execSandboxed()`. When `sandbox:deactivate` calls `killAll()`, the sessions Map is always empty. Server processes become zombies.

**Status**: NOT FIXED — requires architectural change (switch from `exec` to `spawn` for servers + return `ChildProcess` reference). **Accepted risk** for v1: YOLO mode is experimental, single-user, and `processManagerService.killGlobal()` is called on app quit.
